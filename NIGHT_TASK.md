# NIGHT_TASK.md — 유루나비 야간 작업 지시서 (오케스트레이터용)

> 이 파일은 오케스트레이터(메인 세션)가 읽고 실행한다.
> 실행 모드: `claude --permission-mode auto` (일반 사용자 계정, root 아님)
> 너는 오케스트레이터다. 직접 코딩하지 말고 flutter-coder / rust-coder 에게 위임하고,
> 끝나면 code-auditor 에게 검사시킨다. 계획·판단·커밋은 네가 한다.

## 0. 오늘 밤의 미션 (한 줄)

**깨진 경로 탐색을 먼저 복구하고, 그 다음 아래 모듈 큐를 우선순위 순서대로 최대한 진행한다.**
12시간을 채우는 게 목표가 아니다. **한도가 끊기기 전에, 위에 있는 것부터** 끝내는 게 목표다.

---

## 1. 절대 규칙 (모든 모듈에 적용)

1. **한 모듈씩.** 모듈 하나를 완전히 끝내고(완료 기준 충족 → 감사 PASS → 커밋) 다음으로 간다.
   여러 모듈을 동시에 열어 휘젓지 마라. (지난번에 그래서 경로가 깨졌다.)
2. **모듈 시작 전 반드시 git 체크포인트 커밋.** 끝나면 또 커밋. 도중에 끊겨도 잃는 게 없도록.
3. **각 모듈의 "건드리지 말 것" 목록을 엄수.** 범위 밖 파일은 열지도 마라.
4. **검증 없이는 PASS 없음.** 코드를 고치면 `flutter analyze`(Rust면 `cargo build`/`cargo test`)를 돌려
   통과를 확인하고 code-auditor 에게 넘긴다.
5. **막히면 3회까지만 시도.** 3회 안에 감사 PASS를 못 받으면 그 모듈을 **건너뛰고**
   MORNING_REPORT 에 "막힘: 이유"를 적고 다음 모듈로 간다. 무한 루프 금지.
6. **비밀값(API 키) 하드코딩 금지.** .env 사용.
7. **푸시 금지.** 로컬 커밋만. remote push 하지 마라.
8. **토큰 절약(Pro 한도):** 큰 파일을 반복해서 통째로 다시 읽지 마라(필요한 부분만).
   코더는 가능한 한 작은 변경만. 불필요한 리팩터링 금지.

## 2. 모듈 큐 (이 순서대로)

> 각 모듈: ① 체크포인트 커밋 → ② 코더에게 위임 → ③ 검증 → ④ 감사 → ⑤ PASS면 커밋 → 다음.

### ▣ 모듈 0 — 베이스라인 확보 & 진단 (먼저)

- 목표: 현재 상태를 안전하게 고정하고, "경로 탐색이 언제/왜 깨졌는지" 진단.
- 할 일:
  - `git status`, `git log --oneline -20` 확인. 현재 상태를 체크포인트 커밋.
  - 지도 타일을 OSM으로 바꾼 커밋을 git log에서 찾아라. 그 변경이 경로 탐색을 깨뜨렸는지 확인.
  - 판단: (a) 그 커밋만 되돌리고 타일 변경을 올바르게 재적용 vs (b) 현재 코드에서 앞으로 고치기.
    어느 쪽이 안전한지 MORNING_REPORT에 근거와 함께 기록.
  - `flutter analyze` 가 통과하는지 확인(현재 상태 기준).
- 완료 기준: 진단 결론이 MORNING_REPORT에 적힘 + 깨끗한 체크포인트 커밋 존재.
- 건드리지 말 것: 아직 코드 수정 없음. 조사·진단·커밋만.

### ▣ 모듈 1 — 회귀 복구 (★최우선 / 반드시 성공시킬 것)

- 목표: 지도 타일을 OSM으로 유지하면서, 지난번 깨진 기능들을 모두 되살린다.
- 되살릴 기능:
  1. **경로 탐색** — 목적지를 찍으면 3개 코스 카드(시골길/지방도로/국도)가 각각 경로를 그린다.
  2. **목적지 편의점 스냅** — 목적지를 찍으면 주변 편의점/카페에 붙는 동작.
  3. **목적지 반투명 원** — 현위치~목적지 거리를 반투명 원으로 표시.
- 완료 기준(전부 충족):
  - 지도 타일 = OSM (Carto 아님)
  - 목적지를 찍으면 3개 카드 각각 실제 도로 경로가 그려진다
  - 편의점 스냅 동작
  - 반투명 원 표시 동작
  - `flutter analyze` 통과
- 건드리지 말 것: 내비게이션 주행 화면, 테마 색상, 일출일몰 인디케이터.
- 커밋 메시지 예: `fix: restore routing/snap/radius-circle on OSM tiles`

> 모듈 1까지 성공하면 오늘 밤은 "본전 이상"이다. 아래는 시간/한도가 허락하는 만큼 보너스.

### ▣ 모듈 2 — 테마(색상) 독립 모듈화  [사용자 명시 요청]

- 목표: 전 화면의 색을 한 곳(`lib/core/theme`)에서 관리. 화면마다 톤이 다른 문제의 근본 해결.
- 할 일:
  - 테마를 독립 모듈로 정리: **낮(BMNT~EENT) / 밤(EENT~익일 BMNT) / 라이더(고대비)** 팔레트 정의.
  - 내비게이션 화면의 지도가 검게/다른 톤으로 나오는 원인 = 테마 미적용. 테마 모듈을 적용해 해결.
- 완료 기준:
  - 테마 파일 한 곳만 바꾸면 모든 화면(메인/경로/내비) 색이 일관되게 바뀐다.
  - 내비게이션 화면 지도가 더 이상 검지 않고, 메인 화면과 같은 톤으로 보인다.
  - `flutter analyze` 통과.
- 건드리지 말 것: 경로 로직, 지도 타일 URL, 위치 권한 설정.
- 커밋 메시지 예: `refactor: extract theme into independent module (day/night/rider)`

### ▣ 모듈 3 — 지도 표시 다듬기

- 목표: 라이더에게 맞게 지도를 덜 복잡하게, 화면을 정돈.
- 할 일:
  - 줌 단계별 표시 요소 조정(네이버 내비 느낌): 고속도로 과도한 강조 제거, 한적한 길 위주로.
  - 상태표시줄(status bar) 침범 방지(SafeArea 처리).
  - 로고 위치/디자인 정돈(흰색으로 덩그러니 떠 있는 문제).
  - 나침반·줌인·줌아웃 정렬 정리 + **나침반 탭 → 내 위치로 복귀** 동작 복구.
- 완료 기준: 줌별로 지도가 덜 복잡 + 상태바 안 가림 + 로고 정돈 + 나침반 탭하면 내 위치로 이동.
- 건드리지 말 것: 경로 로직, 모듈 2에서 정한 테마 색상값(가져다 쓰기만).
- 커밋 메시지 예: `feat: zoom-level map layers + safe-area + recenter on compass`

### ▣ 모듈 4 — 내비게이션 화면 UI 버그

- 목표: 주행 화면의 명백한 버그들 수정.
- 할 일:
  - **목적지가 없으면 내비 모드 진입 차단**(가드). 지금은 목적지 없이도 작동/안내함.
  - "10초 후 현위치 복귀" 안내가 실제로 복귀하도록 수정.
  - 속도계를 **위쪽**으로 이동(현재 아래에서 ¼ 잘림).
  - 줌/현위치 아이콘이 하단 도착정보에 가리지 않게 위치/순서 조정.
- 완료 기준: 목적지 없으면 내비 안 켜짐 + 속도계 위 + 아이콘 안 가림 + 복귀 동작함.
- 건드리지 말 것: 경로 로직, 모듈 2 테마 색상값.
- 커밋 메시지 예: `fix: navigation screen guards, speedometer position, icon z-order`

### ▣ 모듈 5 — 일출일몰 인디케이터 통일

- 목표: 화면마다 모양이 다른 인디케이터를 공용 위젯 하나로 통일 + 밤 색반전.
- 할 일:
  - 공용 위젯 하나로 통일(메인/내비 동일 모양).
  - 밤(EENT~BMNT)에는 색 반전(예: 낮=해, 밤=달/어두운 톤). 모듈 2 테마와 연동.
  - 위치/가독성 개선, 상태바에 박히지 않게.
- 완료 기준: 모든 화면에서 동일한 인디케이터 + 밤이면 밤모드 색 + 잘 읽힘.
- 건드리지 말 것: 경로 로직, 내비 진입 로직.
- 커밋 메시지 예: `refactor: unify daylight indicator widget + night inversion`

## 4. 아침 보고서 (MORNING_REPORT.md) — 반드시 작성

퇴근한 사용자가 5분 안에 검토할 수 있게, 아래를 한국어로:

1. **완료한 모듈** 목록 + 각 커밋 해시.
2. **각 모듈 완료 기준 충족 여부** (체크 표시).
3. **막힌 것/건너뛴 것** + 이유.
4. **사용자가 폰에서 직접 확인할 체크리스트** (예: "목적지 찍어서 3카드 경로 뜨는지", "내비 화면 색 정상인지").
5. **다음 밤에 이어서 할 것** 제안.
6. **토큰/한도 메모**(언제쯤 한도로 멈췄는지 등, 알 수 있으면).
7. 코드를 못 읽는 사용자를 위해 **전문용어는 풀어서** 설명.
